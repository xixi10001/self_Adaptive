# Post-target 训练机制修改记录

- 修改时间：2026-07-14 15:55:34（Asia/Shanghai）
- 修改目标：在不破坏原 Stage 1-3 和 60% 成功率指标的前提下，增加专门面向 60% 以上覆盖能力的续训流程。
- 初始模型：优先使用 Fixed LR 的 `ppo_best_full_coverage.pth`。

## 修改文件

1. `environment_variables/environment_variables/rl_environment_baseline.py`
2. `environment_variables/environment_variables/ctde_ppo_baseline_train.py`
3. `environment_variables/environment_variables/test_post_target_training.py`（新增）

## 环境修改

新增独立终止模式 `post_target_train`：

- 达到 60% 后不提前终止，继续运行到 `max_steps=600`。
- `target_stop` 的原终止和奖励逻辑保持不变。
- `full_horizon` 继续作为纯评估模式，不注入 post-target 训练奖励。
- 电量耗尽优先按 `battery_depleted` 处理，不允许获得正向 tail 奖励。

新增团队奖励：

- 首次达到 60%：`20 + 10 * (1 - first_target_step / max_steps)`。
- 首次达到 70%：`+5`。
- 首次达到 80%：`+10`。
- 每个里程碑每回合只发放一次，覆盖率波动后不会重复领取。
- 达到 60% 后，普通步惩罚从 `-0.08` 调整为 `-0.02`。
- 保持奖励：`0.10 * clip((coverage - goal) / (1 - goal), -1, 1)`。
- 正常走满时使用最后 100 步平均覆盖率结算：`20 * clip((tail100 - goal) / (1 - goal), -1, 1)`。
- 从未达到 60% 时只使用原超时型失败惩罚，不与负向 tail 重复叠加。

新增奖励分解字段：

- `r_milestone`
- `r_hold`
- `r_tail`

## 续训流程

新增独立入口 `train_post_target()`，与原课程训练主循环分离：

- 首次从普通检查点启动时只加载 Actor/Critic 权重。
- 使用全新的 Adam 优化器，避免旧短轨迹动量影响新目标。
- 从 post-target 检查点恢复时，同时恢复优化器、更新次数、目标阶梯和连续验证计数。
- Actor 输入仍为可部署的 17 维局部观测，没有增加特权信息。
- 环境仍使用 `curriculum_stage=3`，训练阶段在日志中标记为 `post_target`。

前 15 次 PPO 更新采用 warmup：

- Actor LR：`5e-5 -> 1e-4`。
- Critic LR：`1.25e-4 -> 2.5e-4`。
- `clip_epsilon=0.15`。
- `max_grad_norm=0.25`。
- warmup 结束后恢复 `clip_epsilon=0.20`、`max_grad_norm=0.50`。

## 动态目标

保持目标按以下阶梯单向递进：

`0.60 -> 0.65 -> 0.70`

每 100 回合验证一次，必须连续两次通过门槛才升级。

从 0.60 升到 0.65：

- `tail100 >= 0.62`
- `hold_ratio@0.60 >= 0.55`
- `boundary_found_rate >= 0.95`

从 0.65 升到 0.70：

- `tail100 >= 0.67`
- `hold_ratio@0.65 >= 0.50`
- `reach_rate@0.70 >= 0.45`
- `boundary_found_rate >= 0.95`

80% 只作为奖励里程碑，不作为当前训练目标。

## 检查点和模型筛选

新增模型文件：

- `ppo_post_target_ep0100.pth` 等周期检查点。
- `ppo_post_target_final.pth`。
- `ppo_best_post_target.pth`（只有候选模型满足硬门槛时才生成）。

候选模型硬门槛：

- `boundary_found_rate >= 0.95`
- `reach_rate@0.60 >= 0.60`
- `battery_depleted_rate == 0`

合格候选按以下顺序选择：

1. Validation `tail100`。
2. 当前覆盖率 AUC。
3. 历史火线并集覆盖率。

如果没有候选合格，则保留源 `ppo_best_full_coverage.pth`，不生成虚假的最佳 post-target 模型。
