# 火场地图创建与项目集成PPT扩充设计

## 目标

在保留现有模板、配色、页眉和截图风格的基础上，将现有19页PPT扩充为24页。新增内容重点解释项目实际使用的数据、物理意义、对火势的影响以及进入强化学习环境后的用途。

## 内容范围

PPT只介绍以下内容：

1. FlamMap与FARSITE在项目中的职责。
2. 地图创建和火场模拟所需的静态、动态输入数据。
3. FARSITE关键模型参数。
4. 七个火行为栅格和一个火场边界输出。
5. 输出文件加入项目后的索引、加载、预处理和派生数据流程。
6. 火场数据在多无人机强化学习环境中的使用。

不增加训练算法、网络结构、实验结果或与地图流程无关的内容。

## 数据解释标准

每种数据尽量包含四部分：

- 数据定义
- 物理意义
- 对火势或模拟结果的影响
- 在当前项目中的实际用途

静态地图明确说明8个波段：海拔、坡度、坡向、燃料模型、林冠覆盖度、林冠高度、林冠基部高度和林冠容积密度。

动态输入明确说明燃料湿度、逐时气象、点火位置、障碍物和燃烧时段。

输出数据明确说明到达时间、火焰长度、蔓延速度、火线强度、蔓延方向、单位面积热量、冠火活动和火场边界。

## 项目事实与修正

- 示例静态地图分辨率为30 m，每个像元面积为900平方米。
- `.fms` 文件最后一列按FlamMap格式解释为1000小时燃料湿度，不解释为叶片湿度。
- 叶片湿度来自独立的 `Foliar Moisture Content` 参数，示例值为100%。
- 当前蔓延方向栅格值域约为0到6.28，按弧度解释。
- 项目将FARSITE输出映射为 `intensity`、`time`、`length`、`speedRate`、`spread_direction`、`heat_per_unit_area` 和 `crown_fire`。
- 项目使用火线强度和到达时间生成动态火场，进一步计算火场边界、热场和综合严重度。
- 综合严重度权重为：火线强度0.35、火焰长度0.20、蔓延速度0.20、单位面积热量0.15、冠火活动0.10。

## 版式策略

- 保留现有模板的页眉、蓝色分隔线、字体层级、配色和页码风格。
- 封面保持简洁。
- 数据解释页优先使用表格、字段示例和少量说明，不使用大段正文。
- 原始界面截图继续用于输入、参数和输出选择页面。
- 同一页面只表达一个主要结论，避免为了增加内容而压缩字号。
- 技术定义页使用简短来源脚注，项目实现页以本地代码和数据文件为依据。

## 验证标准

1. 最终文件包含24页。
2. 模板页眉、页脚和配色保持一致。
3. 页面无文字溢出、非预期重叠或截断。
4. `.fms` 最后一列和蔓延方向单位使用正确解释。
5. 文件字段、目录结构和代码字段与当前项目一致。
6. 全部页面完成渲染检查，并通过演示文稿结构检查。
