# 火场地图PPT图片增强设计

## 目标

以现有 `森林火场地图创建与项目集成_详细版.pptx` 为源文件，创建独立副本 `森林火场地图创建与项目集成_图片增强版.pptx`。不覆盖现有文件。

删除原第9页“静态地图（二）：四类林冠结构数据”，最终PPT为23页。其余页面保持现有叙事顺序，同时增加项目真实数据图、软件截图和少量流程示意图。

## 图片原则

- 项目真实数据和真实文件截图优先。
- 官方FlamMap、FARSITE和LANDFIRE图片作为软件与数据来源说明。
- 只在项目背景和强化学习使用页使用概念示意图。
- 不制作虚假的软件界面或伪造项目结果。
- 每张图片必须承担解释功能，不以装饰性火灾照片填充页面。

## 选择FlamMap的说明

PPT中补充以下选择依据：

- 支持项目使用的标准景观数据结构。
- 能统一管理地形、燃料、湿度和气象数据。
- 输出保留空间坐标，便于以GeoTIFF和Shapefile接入项目。
- 避免自行重新实现完整火行为计算模型。
- FlamMap 6集成FARSITE，软件链路统一。

## 选择FARSITE的说明

PPT中补充以下选择依据：

- 项目需要动态火场，而不是静态危险度图。
- 支持点火区域、逐时天气、燃烧时段和空间分辨率。
- 能输出到达时间、蔓延速度、火线强度和火场边界。
- `arrival_time` 可以直接驱动强化学习环境中的火区动态更新。

## 使用JSON接入环境的说明

JSON只作为索引和元数据控制层，不保存大型地图数组。

- GeoTIFF保存栅格。
- Shapefile保存点火和边界几何。
- FMS/WXS保存燃料湿度和天气。
- JSON保存路径、场景编号、数据集划分、参数和统计信息。

选择JSON是因为它支持嵌套结构、Python原生读取、人工可检查、便于版本管理，且比数据库更适合当前本地文件型数据集。
