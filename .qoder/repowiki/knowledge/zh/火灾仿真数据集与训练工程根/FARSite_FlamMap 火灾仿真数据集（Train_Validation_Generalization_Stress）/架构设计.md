纯数据仓库型目录结构，无代码实现，按「任务类型 → 区域编号 → 具体场景」三层树形组织：
- 顶层 `Train/1..4`、`Validation/5`、`Generalization/6,7`、`Stress/8` 对应不同学习/评估阶段；每个区域目录下包含一个 `mapN/` 子目录（存放该区域的 `mapN.tif` 地形栅格及同名 `.fmp/.fzp` FlamMap 工程文件）。
- 每个 `sceneK` 是一个独立 FARSite 运行实例，内部统一划分为四个子目录：`inputs/`（`.fms` 燃料湿度 + `.wxs` 气象流）、`rasters/`（`arrival_time.tif`、`ros_farsite.tif`、`flame_length_farsite.tif`、`fireline_intensity_farsite.tif`、`spread_direction_farsite.tif`、`heat_per_unit_area_farsite.tif`、`crown_fire_activity_farsite.tif` 等输出栅格）、`vectors/`（`ignition.shp` 点火源 + `fire_perimeter.shp` 火场边界，配套 `.dbf/.prj/.shx`）、`reports/`（`Run_log.txt` + `fire_growth_report.csv`）。
- 每个 `sceneK` 根级附带 `metadata.json`，以结构化方式记录 `map_id`、`split`、`area_id`、`scenario_id`、分辨率、风场参数、燃料湿度类、FARSite 运行配置、目标范围与实际统计、UAV 参数及各数据文件的相对路径，作为上层加载器的唯一契约入口。
- 依赖方向单向：上层 Python 模块仅读取本目录下的 GeoTIFF、Shapefile、CSV/TXT 与 JSON 元数据，不修改任何原始数据；每个区域还打包一份 `mapN.zip` 便于分发。
